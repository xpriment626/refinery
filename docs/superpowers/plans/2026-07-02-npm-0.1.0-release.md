# Refinery 0.1.0 Npm Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prepare Refinery for a first public npm release as `@itsshadowai/refinery@0.1.0` with a bundled Codex skill and a `refinery init` onboarding command.

**Architecture:** Keep the CLI as a TypeScript/Node package that ships compiled `dist/` JavaScript plus runtime Coral agent manifests and a packaged Codex skill under `skills/refinery`. `refinery init` creates global Refinery state under `~/.refinery` and installs the packaged skill into `${CODEX_HOME:-~/.codex}/skills/refinery` without overwriting user edits unless `--force` is passed.

**Tech Stack:** Node.js >= 22, TypeScript, npm package `bin`, Codex skills, Coral local agent manifests.

## Global Constraints

- Package name must be `@itsshadowai/refinery`.
- Binary command must remain `refinery`.
- Version must be `0.1.0`.
- License must be MIT.
- The npm tarball must include runtime files only: `dist`, `coral`, `skills`, `README.md`, `LICENSE`, and `package.json`.
- The bundled Codex skill must be available as `$refinery`.
- `refinery init` must preserve existing installed skills unless `--force` is passed.
- Publishing itself stays manual and interactive because npm 2FA is enabled.
